## Versioning Scheme
- Major version (X.0.0): Breaking changes
- Minor version (0.X.0): New features

## Release Process
1. Update version numbers
2. Update changelog
3. Create release branch
4. Perform final testing
5. Merge to main
6. Tag release
7. Deploy to production
